Risk Adjustment Coding-Accuracy Tool

Compare a submitted set of ICD-10 codes against a gold/validated set, map both to HCC categories (CMS-HCC V28 sample), and measure coding accuracy with precision, recall, F1, and false-positive / false-negative breakdowns — at both the ICD-10 and HCC levels.
Why this matters (the 20-second version): Medicare pays Medicare Advantage plans more for sicker members. Each diagnosis (ICD-10) can map to an HCC category that carries payment weight. A missed valid code (false negative) = lost revenue and an incomplete risk picture. An unsupported submitted code (false positive) = audit/compliance risk and potential overpayment. A real DOJ settlement in 2026 penalized a program that fixed only the missed codes and ignored the unsupported ones — which is exactly why this tool reports both error types.

What the numbers mean

Precision = TP / (TP + FP). Of everything you submitted, how much was actually supported. Low precision = over-coding = audit/compliance risk.
Recall = TP / (TP + FN). Of everything you should have captured, how much you got. Low recall = under-coding = lost revenue and an incomplete risk picture.
F1 = 2·P·R / (P + R). The harmonic mean — only high when both precision and recall are high. It can't be gamed by being great at one and terrible at the other (precision 1.0, recall 0.0 → F1 = 0, not 0.5).
Code level vs HCC level. Two different codes can map to the same HCC. E11.22 vs E11.21 is a code-level FP+FN, but if both map to the same HCC it's a True Positive at the HCC level — and the HCC level is what actually drives the payment. Showing both is the point.
